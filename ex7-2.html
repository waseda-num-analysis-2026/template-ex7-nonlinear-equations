Exercise 7.2 — Comparing Root-Finding Methods

Numerical Analysis 2026 — Lecture 7

Problem

For

\[ f(x)=x-\cos x=0 \]

(unique real root \(\approx 0.7391\)), implement and compare:

  1. Newton’s method — use \(f'(x_k)\) at each step
  2. Simplified Newton’s method — use \(f'(x_0)\) only once
  3. Bisection method — halve a bracketing interval

Tasks:

  • Compare the number of iterations, function/derivative evaluations, convergence rate, and robustness
  • Plot the error history (AI permitted; plots recommended)

AI is permitted. AI is recommended for plotting.

1. Target function

import numpy as np
import matplotlib.pyplot as plt
import time

def f(x):
    return x - np.cos(x)

def df(x):
    return 1 + np.sin(x)

x_star = 0.7390851332151607  # reference root (high accuracy)

def newton_method(f, df, x0, max_iter=100, tol=1e-12):
    x_history = [x0]
    f_evals = 0
    df_evals = 0
    start_time = time.time()
    x = x0

    for k in range(max_iter):
        fx = f(x)
        dfx = df(x)
        f_evals += 1
        df_evals += 1
        x_new = None  # TODO
        x_history.append(x_new)

        if abs(fx) < tol:
            end_time = time.time()
            return {
                "solution": x_new,
                "iterations": k + 1,
                "function_evaluations": f_evals,
                "derivative_evaluations": df_evals,
                "total_evaluations": f_evals + df_evals,
                "history": np.array(x_history),
                "converged": True,
                "execution_time": end_time - start_time,
                "method": "Newton",

        x = x_new

    end_time = time.time()

    }

4. Bisection method (template provided)

def bisection_method(f, a, b, max_iter=100, tol=1e-12):
    x_history = []
    f_evals = 2
    start_time = time.time()
    fa = f(a)
    fb = f(b)

    if fa * fb >= 0:
        end_time = time.time()
        return {
            "solution": None,
            "iterations": 0,
            "function_evaluations": f_evals,
            "derivative_evaluations": 0,
            "total_evaluations": f_evals,
            "history": np.array([]),
            "converged": False,
            "execution_time": end_time - start_time,
            "method": "Bisection",
            "error": "Initial interval does not bracket a root",
        }

    for k in range(max_iter):
        c = (a + b) / 2
        x_history.append(c)
        fc = f(c)
        f_evals += 1

        if abs(fc) < tol or (b - a) / 2 < tol:
            end_time = time.time()
            return {
                "solution": c,
                "iterations": k + 1,
                "function_evaluations": f_evals,
                "derivative_evaluations": 0,
                "total_evaluations": f_evals,
                "history": np.array(x_history),
                "converged": True,
                "execution_time": end_time - start_time,
                "method": "Bisection",
            }

        if fa * fc < 0:
            b = c
            fb = fc
        else:
            a = c
            fa = fc

    end_time = time.time()
    return {
        "solution": (a + b) / 2,
        "iterations": max_iter,
        "function_evaluations": f_evals,
        "derivative_evaluations": 0,
        "total_evaluations": f_evals,
        "history": np.array(x_history),
        "converged": False,
        "execution_time": end_time - start_time,
        "method": "Bisection",
    }

Use a common initial guess (for example \(x_0=1.0\)) and a valid bracket \([a,b]\) for bisection (for example \([0,1]\)).

Why count function and derivative evaluations? A method with fewer iterations is not necessarily cheaper overall. Newton may converge in very few steps, but each step evaluates both \(f\) and \(f'\). Simplified Newton evaluates \(f'\) only once. Bisection uses only \(f\), but often needs more iterations. Recording these counts helps you compare convergence speed (how many steps) with work per step (how many function values are computed).
